2. 暂时性死区（TDZ - Temporal Dead Zone）

在声明前访问let/const变量会触发错误（而var返回undefined）：

TDZ 本质：
从进入作用域到变量声明之间的区域，禁止访问变量

3. const 的特殊行为

const 本质：

  • 保证变量指向的内存地址不变（对于对象是堆地址，对于基本类型是栈值）
  • 对象属性修改不受限制（除非使用Object.freeze）

最佳实践指南

  1. **默认使用 const**
    除非需要重新赋值，否则优先用const（减少意外修改）

  2. **需要重新赋值时用 let**
    替代var的所有场景

  3. **避免使用 var**
    除特殊兼容场景外不应使用
