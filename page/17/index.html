原因：
JavaScript 早期设计时用二进制前三位标识类型：

  • 000：对象类型
  • 1：整型
  • 010：双精度浮点型
  • 100：字符串
  • 110：布尔型

5. typeof NaN 的特殊性

NaN 的特性：

  • 全称：Not-A-Number（非数字）
  • 是特殊的 Number 类型值
  • 由无效数学运算产生：0/0, Math.sqrt(-1)
  • 不等于任何值（包括自身）：NaN === NaN → false

TDZ 本质：
从进入作用域到变量声明之间的区域，禁止访问变量

3. const 的特殊行为

const 本质：

  • 保证变量指向的内存地址不变（对于对象是堆地址，对于基本类型是栈值）
  • 对象属性修改不受限制（除非使用Object.freeze）

最佳实践指南

  1. **默认使用 const**
    除非需要重新赋值，否则优先用const（减少意外修改）

  2. **需要重新赋值时用 let**
    替代var的所有场景

  3. **避免使用 var**
    除特殊兼容场景外不应使用
